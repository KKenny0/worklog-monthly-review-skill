# Daily Note 格式约定

本文档记录 Daily Note.md 的实际格式规范，供解析脚本和 Claude 参考。

## 标题层级结构

Daily Note 使用三级标题组织内容：

```markdown
# 2026 年                          ← 年标题（一级标题）
                                   ← 空行

## 2026 年 3 月                    ← 月标题（二级标题）
### 2026.03.02                     ← 日标题（三级标题）
...内容...
### 2026.03.03
...内容...

## 2026 年 4 月
### 2026.04.01
...内容...
```

### 正则匹配规则

| 层级 | 格式 | 正则 |
|-----|------|------|
| 年 | `# YYYY 年` | `^#\s*\d{4}\s*年` |
| 月 | `## YYYY 年 M 月` | `^##\s*\d{4}\s*年\s*\d{1,2}\s*月` |
| 日 | `### YYYY.MM.DD` | `^###\s*\d{4}\.\d{2}\.\d{2}` |

## 内容结构

每个日期下的内容按如下结构组织：

```markdown
### 2026.03.02
- [项目名]
	- {模块名}
		- 当前应用：应用名
		- 今日进展：
			- 【类别标签】
				- [x] 已完成的任务描述
					- 详细说明行
				- [ ] 未完成的任务描述
			- 【阶段总结】
				- 总结描述
```

### 项目标签格式

```markdown
- [项目A]
- [项目B]
- [专利]
```

一天内可以有多个项目。

### 模块标签格式

```markdown
	- {模块A}
	- {模块B}
	- {模块C}
```

模块名使用花括号 `{}`，前面有一个 Tab 缩进。

### 模块组合标签

有时模块标签会包含多个标签：

```markdown
	- {模块A}-{模块B}
```

### 类别标签格式

使用中文方括号 `【】` 标记：

```markdown
			- 【能力升级】
			- 【结构变更】
			- 【问题定位】
			- 【配置调整】
			- 【文档优化】
			- 【阶段总结】
			- 【阶段冻结】
```

### 任务状态标记

```markdown
- [x] 已完成    ← 方括号内有 x
- [ ] 未完成    ← 方括号内有空格
```

### 代码变更量标注

部分条目会标注代码变更行数：

```markdown
- [x] 功能集成重构（+1952/-735 行）
```

### 链接格式

内部文档链接使用 Markdown 标准格式：

```markdown
[文档标题](URL)
[文档标题](本地路径)
```

### TODO LIST

文件末尾可能有 TODO LIST 区域，不属于任何月份：

```markdown
【TODO LIST】
- [项目A · 子系统]
	- {模块X}
		- [ ] 待办事项
```

## 格式变体说明

### 无"今日进展"标签的条目

部分简短条目可能省略"今日进展："标签，直接在模块下写内容：

```markdown
### 2026.03.05
- [项目B]
	- {模块D}
		- [x] 生产服务竞态死锁问题修复
```

### 多项目交叉

同一天可能有多个项目的工作记录，每个项目使用独立的 `- [项目名]` 标记：

```markdown
### 2026.03.09
- [项目A]
	- {模块A}
		...
- [项目B]
	- 测试环境部署问题定位 & 修复
	- 模块联调支持
```

### 简短记录

有些条目非常简短，没有嵌套的详细说明：

```markdown
### 2026.03.11
- [项目A]
	- {模块A}
		- {子模块交付}
```

## 缩进规范

使用 Tab 缩进（非空格），每级一个 Tab：

```
Level 0: - [项目名]
Level 1: \t- {模块名}
Level 2: \t\t- 内容标签
Level 3: \t\t\t- 详细条目
Level 4: \t\t\t\t- 子条目
```
