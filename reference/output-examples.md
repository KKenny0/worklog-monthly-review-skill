# 输入输出样例

本文档提供 Daily Note 到月度归档和月度总结的输入输出样例，帮助对齐预期。

## 输入样例：Daily Note.md 片段

```markdown

## 2026 年 3 月
### 2026.03.02
- [项目A]
	- {模块A}
		- 当前应用：产品X
		- 今日进展：
			- 【结构变更】
				- [x] 本地项目方案设计
					- 整体架构设计：层级结构、数据流转流程等
					- 数据模型设计
					- 长批处理策略设计
				- [x] 初版方案开发
			- 【问题定位】
				- [ ] Pipeline 本地测试与 Bug 修复
					- PipelineRunner + StageRunner 执行链路测试 & Debug
			- 【阶段总结】
				- 整体架构搭建完成
				- 正在集中处理组装阶段数据流转的细节问题
### 2026.03.03
- [项目A]
	- {模块A}
		- 当前应用：产品X
		- 今日进展：
			- 【问题定位】
				- [x] Pipeline 本地测试与 Bug 修复
					- PipelineRunner + StageRunner 执行链路测试 & Debug
			- 【能力升级】
				- [x] 核心功能模块开发调整
					- 生成模块实现异步批处理与并发控制
				- [x] 文本解析增强
					- 支持中英文标点正则匹配
### 2026.03.05
- [项目A]
	- {模块A}
		- 当前应用：产品X
		- 今日进展：
			- 【能力升级】
				- [x] 角色管理增强
				- [x] 本地流程组装重构
				- [x] LLM API 使用量追踪系统

- [项目B]
```

- 包含月标题和日标题

## 输出样例 2：月度总结 2026-03.summary.md

```markdown
# 2026-03 工作总结

> 统计：3 个工作日（样例片段）| 2 个项目 | 8 项已完成 | 1 项进行中

## 本月主线

- **项目A**（3 天活跃） — 自动化系统从方案设计到核心功能开发，覆盖核心流程各模块
- **项目B**（1 天活跃） — 生产服务稳定性修复

## 高频主题

- 核心生成模块
- 角色管理
- Pipeline
- 文本解析
- LLM
- 异步批处理

## 重点推进事项

### 项目A

本月（样例片段）主要推进了自动化系统的从 0 到 1 建设：

- [2026.03.02] 完成项目方案设计，包含整体架构、数据模型和批处理策略
- [2026.03.03] Pipeline 本地测试与核心功能模块开发，实现异步批处理
- [2026.03.05] 角色管理和本地流程组装的重构，引入 LLM 使用量追踪

### 项目B

- [2026.03.05] 生产服务竞态死锁问题修复

## 已完成

### 项目A

- [2026.03.02] 项目方案设计
- [2026.03.02] 初版方案开发
- [2026.03.03] Pipeline 本地测试与 Bug 修复
- [2026.03.03] 核心功能模块开发调整（异步批处理与并发控制）
- [2026.03.03] 文本解析增强（中英文标点支持）
- [2026.03.05] 角色管理增强
- [2026.03.05] 本地流程组装重构
- [2026.03.05] LLM API 使用量追踪系统

### 项目B

- [2026.03.05] 生产服务竞态死锁问题修复

## 进行中

### 项目A

- [2026.03.02] Pipeline 本地测试与 Bug 修复（部分完成，待验证）
- [2026.03.05] 长流程测验 & 调整中

## 阻塞与风险

- [2026.03.02] [项目A] Pipeline 执行链路测试存在 Bug 待修复

## 建议的下一步

- [项目A] 继续推进长流程的验证与调整
- [项目A] 解决 Pipeline 执行链路中剩余的 Bug

## 信息缺口说明

- 样例仅包含 3 天数据，完整月份的工作覆盖范围更广
```

**关键特征：**
- 每条已完成事项都有 `[x]` 标记对应
- 进行中事项如实标注来源状态
- 风险和建议都基于原始记录，不凭空扩写
- 信息缺口如实说明
